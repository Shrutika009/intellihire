Two-Stage Retrieval Framework

To run the heavy Sentence Transformer models under the 5-minute CPU constraint, we implement a production-grade two-stage framework:

Stage 1 (Retrieval): We parse all 100,000 candidates and perform a lightweight scoring pass on explicit features (skills, YOE, availability signals). We select the top 2,000 candidates.

Stage 2 (Re-ranking): We run our Sentence Transformer multi-qa-mpnet-base-dot-v1 model only on the top 2,000 candidate narratives to re-rank and pick the final top 100.

Formula Specification

final_score = (
  0.40 * semantic_sim +
  0.25 * skill_match +
  0.18 * experience_composite +
  0.12 * availability +
  0.05 * trust
) * inflation_multiplier

Inflation Penalty

If a candidate lists skills but less than 30% of them appear in their actual career summaries/role histories, the algorithm applies a 0.7× multiplier penalty to neutralize keyword stuffers.

Challenge Blueprint

The job description details, requirements, and validation checklist.

Job Description Requirements

Hard Core Filters

5-9 years experience, Production embeddings-based retrieval systems (sentence-transformers, OpenAI embeddings, BGE, E5), Vector DBs (Pinecone, Weaviate, Qdrant, Milvus, Elasticsearch, FAISS), evaluation frameworks design (NDCG, MRR, MAP).

Soft Requirements

LLM fine-tuning (LoRA, QLoRA, PEFT), Learning-to-rank models, distributed systems, HR-tech or marketplace product experience.

Disqualifiers & Red Flags

Pure research background with no shipping, LangChain-only hobby projects without pre-LLM ML depth, no production code in 18+ months, title chasers (switching companies every 1.5 yrs), only consulting firm experience.

Evaluation Metrics

NDCG@10 dominates the scoring at 50%:

  • NDCG@10 (50%): Heavy focus on the top 10 positions.
  • NDCG@50 (30%): Mid-level ranking relevance.
  • MAP (15%): Overall precision.
  • P@10 (5%): Precision in top 10.

Validation Success Criteria

  • ✓ Exactly 100 data rows in CSV.
  • ✓ Monotonically decreasing scores.
  • ✓ Non-empty, highly specific reasoning strings.
  • ✓ Under 5-minute CPU runtime.
  • ✓ Honeypot count in top-100 is under 10%.
